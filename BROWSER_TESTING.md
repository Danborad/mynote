# 浏览器测试说明

这份文档用于规范 `MyNote` Web 端的人工/自动化浏览器测试范围，避免只验证登录页而遗漏主流程页面。

当前测试账号：

- 用户名：`zhong`
- 密码：`onepiece`

截图目录：`screenshots/`

说明：

- 浏览器测试不能只覆盖登录页
- 必须覆盖登录后的主工作流页面
- 所有关键页面至少保留一张截图作为验证证据
- 截图统一存放到仓库根目录下的 `screenshots/`

## 测试目标

验证 Web 端在真实浏览器中的以下内容：

- 登录流程可用
- 登录后主页可用，布局与主题正常
- 深色模式和浅色模式都没有明显样式错误
- 分享链接、设置、笔记编辑等关键页面可进入且可交互
- 关键弹层、抽屉、菜单不会出现错位、硬边、白块、裁切或异常滚动

以下页面和场景必须测试，不允许只测登录页：

- 登录页成功打开
- 用户名、密码输入框可输入

## 推荐测试顺序

建议按下面顺序执行浏览器测试：

1. 打开登录页
2. 使用 `zhong / onepiece` 登录
3. 截图首页浅色模式
4. 切换深色模式，截图首页深色模式
5. 打开一条笔记，截图编辑器浅色/深色模式
6. 打开设置页，截图浅色/深色模式
7. 打开分享链接页，重点检查边缘和抽屉背景，再截图
8. 打开搜索、更多菜单、确认框等弹层并截图
9. 验证背景颜色修改不影响笔记时间和排序
